## 模块说明

`Content` 模块负责把垃圾分类玩法所需的模型资源接入到主场景中，目标是把“资源文件”转成“可用于 MVP 演示的场景对象”。

当前包含：
- `WasteContentCatalog`：维护垃圾与垃圾桶的资源路径、名称、分类、错误提示和默认生成位置。
- `WasteContentMenu`：提供编辑器菜单，用于创建默认内容目录配置。
- `WasteSceneBuilderWindow`：批量把目录中的资源拉入场景，并自动挂接分类与交互所需组件。

当前约束：
- `Assets/Scenes/2.unity` 是唯一主场景。
- `Assets/Scenes/1.unity` 只作为旧原型参考，不再继续接入新内容。

当前限制：
- Unity 2020.3 默认不能直接把 `.glb` 当作可实例化的原生资源使用。
- 只有在 glTF 导入能力已接入，或者资源已被转换成 Unity 可识别的 `GameObject` 资产后，场景构建工具才能真正实例化这些模型。

推荐流程：
1. 先执行菜单创建默认内容目录配置。
2. 确认当前 Unity 环境已经具备 `.glb` 导入能力，资源可以在 Project 视图中正常导入。
3. 打开 `2.unity`。
4. 执行场景构建工具，一键生成 12 个垃圾和 4 个垃圾桶。
5. 在 Unity 中微调位置、缩放、碰撞范围和触发区。
