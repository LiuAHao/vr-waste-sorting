# ParkClean VR V0.3 MVP 开发计划

## 1. 版本目标

V0.3 的目标是先完成一个可在桌面端稳定运行的垃圾分类 MVP，而不是在这一版内完成 VR 全量适配。

这一版需要做到：

- 玩家能在场景内移动、观察并抓取垃圾
- 玩家能把垃圾投放到四分类垃圾桶
- 系统能判断投放是否正确
- 系统能给出即时反馈、计分和结算
- 当前代码结构能为后续 VR 交互适配保留复用空间

## 2. 当前主干已完成的范围

截至当前 `main` 分支，以下能力已经具备：

- `Gameplay` 层已经具备垃圾数据、垃圾桶分类、投放触发和分类结果事件
- `Interaction` 层已经接入桌面端键鼠移动、抓取、悬停高亮和辅助射线
- `Core/UI` 层已经接入倒计时、得分、进度、即时反馈、结算页和重开
- 主流程已经从“投放结果事件”驱动，不依赖具体桌面输入实现

## 3. MVP 成功标准

本版完成后，至少满足以下条件：

1. 玩家进入场景后可以正常移动和观察。
2. 场景中有四类垃圾桶：可回收物、有害垃圾、厨余垃圾、其他垃圾。
3. 场景中有一组可抓取、可分类的垃圾物体。
4. 每个垃圾物体具备名称、分类和错误原因。
5. 玩家可以用鼠标抓起垃圾并移动到垃圾桶附近。
6. 垃圾进入投放区域后，系统能判断对错。
7. 正确投放会增加进度和得分。
8. 错误投放会扣分、提示原因，并允许重试。
9. 游戏中会显示时间、得分、进度和反馈信息。
10. 完成目标或时间结束后会显示结算页。

## 4. 这一版明确不做的内容

为了保证交付，V0.3 暂不包含以下内容：

- VR 头显和手柄适配
- 手势识别
- 多场景切换
- 复杂投掷物理
- 联网排行榜和数据上传
- 成就、连击、多轮玩法
- 复杂空间 UI 和 VR 交互细节

## 5. 推荐的模块边界

### Gameplay

负责分类规则和投放判定：

- `WasteCategory`
- `GarbageItem`
- `TrashBin`
- `DropZone`
- `ClassificationResult`
- `ClassificationEvents`

### Interaction

负责桌面端输入与抓取：

- `Player`
- `DesktopGarbageInteractor`
- `DesktopInteractionBootstrap`
- `SelectableHighlighter`
- `CrosshairController`

### Core / UI / Analytics

负责任务流程、反馈和结算：

- `WasteGameBootstrap`
- `WasteGameFlowController`
- `WasteHudView`
- `WasteResultView`
- `WasteAnalyticsTracker`

## 6. 当前阶段的收口重点

当前主线已经基本跑通，接下来不建议继续加新功能，而应该优先收口：

1. 修复代码与文档乱码。
2. 清理场景中的测试残留对象。
3. 稳定旧 UI 隐藏逻辑，避免新旧 HUD 叠层。
4. 补充一份可重复执行的 MVP 验收清单。

## 7. 验收建议

每次提交前，至少手动验证以下流程：

1. 启动场景，确认 HUD 正常显示。
2. 抓取一个垃圾并正确投放。
3. 抓取一个垃圾并错误投放。
4. 确认错误后垃圾回到初始位置并可再次抓取。
5. 确认正确投放后不会重复计分。
6. 跑通一次全部完成结算。
7. 跑通一次超时结算。
8. 点击重开，确认数据归零。
