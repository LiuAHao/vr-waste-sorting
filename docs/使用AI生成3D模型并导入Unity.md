# 使用AI生成3D模型并导入Unity

## 1. 这份文档解决什么问题

这份文档面向 Unity 小白，目标是帮助团队用最简单的方式完成这条流程：

`AI 生成 3D 模型 -> 导出模型文件 -> 导入 Unity -> 放进场景 -> 调整可用`

这条路线适合当前项目，因为 Unity 主要负责场景搭建、交互和运行，不适合拿来做专业建模。建模本身更适合交给：

- AI 3D 生成平台
- Blender
- 现成资源包

## 2. 适用范围

方案 A 最适合以下模型：

- 公园垃圾桶
- 长椅
- 路牌
- 树桩
- 花盆
- 石头
- 小型装饰物
- 场景里的静态道具

不建议一开始就用这个方案直接做：

- 主角人物
- 需要精细骨骼动画的角色
- 结构特别复杂的大型建筑
- 对碰撞和性能要求极高的核心模型

## 3. 推荐工具路径

### 路径一：最适合小白

`AI 3D 平台 -> 导出 GLB/FBX -> Unity 导入 -> 必要时微调`

这条路径优点：

- 上手最快
- 不需要先学会 Blender 建模
- 适合快速出原型

### 路径二：稍微专业一点

`AI 3D 平台 -> 导出 GLB/FBX -> Blender 简单清理 -> Unity 导入`

这条路径优点：

- 更容易修正缩放、朝向、材质、模型破面
- 更适合后续正式项目

如果团队成员完全是新手，建议先从路径一开始，遇到问题再补 Blender 的基础操作。

## 4. 推荐平台

可以优先尝试腾讯混元生 3D，或者其他成熟的 AI 3D 生成平台。

推荐原因：

- 支持文生 3D
- 支持图生 3D
- 支持多视图输入
- 可以导出 `GLB`、`OBJ` 等常见格式
- 官方说明可用于 Unity、Unreal、Blender 等工作流

如果只是做静态道具，优先尝试：

- 文生 3D：直接输入文字描述
- 图生 3D：上传参考图生成

## 5. 文件格式怎么选

### 首选：`GLB`

适合：

- 静态模型
- 带贴图的普通道具
- 希望一个文件尽量带全模型和材质

优点：

- 文件集中
- 材质和贴图通常更省事
- 现代实时 3D 工作流常用

注意：

- Unity 默认不一定原生直接把 `GLB` 工作流处理到最顺手
- 本项目当前 `Packages/manifest.json` 中还没有 `glTFast` 包
- 如果要稳定导入 `GLB`，建议安装 `glTFast`

### 备选：`FBX`

适合：

- Unity 传统工作流
- 兼容性优先
- 后续可能接动画的模型

优点：

- Unity 团队更常用
- 导入经验多
- 遇到问题时更容易搜到解决方案

### 不建议优先：`OBJ`

原因：

- 材质和贴图通常分开管理
- 容易出现贴图丢失
- 小白更容易导错

## 6. 实操流程

### 第一步：先确定你要生成什么

不要一上来就做复杂模型，建议先从一个静态小物件练手。

推荐第一批练手目标：

- 公园长椅
- 垃圾桶
- 指示牌
- 草地石块
- 花盆

### 第二步：写提示词

提示词尽量包含以下信息：

- 物体名称
- 风格
- 材质
- 复杂度
- 使用场景

提示词模板：

```text
一个[物体名称]，风格为[写实/卡通/低多边形]，材质为[木头/金属/塑料]，
适合Unity VR项目使用，模型结构简洁，贴图干净，适合实时渲染。
```

示例 1：

```text
一个低多边形公园垃圾桶，绿色塑料材质，卡通写实结合风格，适合Unity VR项目，结构简洁，贴图干净。
```

示例 2：

```text
一个低多边形公园长椅，木质座面，黑色金属支架，适合Unity VR项目使用，实时渲染友好。
```

### 第三步：生成并下载模型

导出时优先尝试以下顺序：

1. `GLB`
2. `FBX`
3. `OBJ`

建议：

- 如果平台同时支持低模或面数控制，优先选择较低面数
- 如果支持贴图分辨率选择，先选中等分辨率
- 不要第一版就追求最高精度

## 7. 导入 Unity 的具体做法

### 做法一：导入 `FBX`

这是最稳妥的基础做法。

操作步骤：

1. 打开 Unity 项目
2. 在 `Assets` 下新建一个资源目录，例如：
   - `Assets/AIModels`
   - `Assets/AIModels/Bench`
3. 把导出的 `FBX` 和贴图文件拖进去
4. 等 Unity 自动导入
5. 把模型拖到场景中查看

### 做法二：导入 `GLB`

如果你想用 `GLB`，建议先安装 `glTFast`。

安装思路：

1. 打开 Unity
2. 进入 `Window > Package Manager`
3. 搜索或添加包 `com.unity.cloud.gltfast`
4. 安装完成后再导入 `GLB`

然后：

1. 在 `Assets` 下建立目录，例如 `Assets/AIModels`
2. 把 `.glb` 文件拖进去
3. 等 Unity 处理导入
4. 拖入场景查看效果

### 当前项目建议

对当前 `ParkClean-VR` 项目，建议优先顺序如下：

1. 先试 `FBX`
2. 如果你们后面想走更现代、更统一的工作流，再补 `glTFast` 用 `GLB`

原因很简单：

- 你们现在是新手
- `FBX` 对 Unity 初学者更直观
- `GLB` 虽然好用，但通常需要先补一下包和工作流认知

## 8. 导入后必须检查的 6 件事

### 1. 缩放是否正常

常见问题：

- 模型太大
- 模型太小

处理方式：

- 先在场景里拖进去看比例
- 再根据需要调整 `Transform > Scale`

### 2. 朝向是否正确

常见问题：

- 模型横着躺
- 模型旋转 90 度

处理方式：

- 先尝试在 Unity 中旋转
- 如果每次导入都不对，后续建议用 Blender 调整后重新导出

### 3. 材质和贴图是否丢失

常见问题：

- 模型全白
- 模型全粉
- 纹理不显示

处理方式：

- 检查贴图文件是否一起导入
- 检查材质球是否自动生成
- 如果是粉色，通常是 Shader 不匹配或材质导入异常

### 4. 面数是否过高

VR 项目非常怕高面数模型堆积。

常见风险：

- 场景卡顿
- 帧率下降
- 头显体验不稳定

建议：

- 同类道具优先使用低模版本
- 不要一次往场景里扔很多高精度 AI 模型

### 5. 底部是否贴地

常见问题：

- 模型悬空
- 模型一半埋进地里

处理方式：

- 在场景中微调位置
- 如果原点不合理，后续可在 Blender 里修正

### 6. 是否需要碰撞体

AI 生成模型通常不自动满足游戏碰撞需求。

如果这个物体需要交互或阻挡，记得加：

- `Box Collider`
- `Mesh Collider`
- `Capsule Collider`

对于 VR 项目，静态简单物体优先用简单碰撞体，不要滥用复杂 `Mesh Collider`。

如果一个模型完整导入后没问题，再做成 Prefab，便于复用。

## 10. 推荐的小白工作顺序

请按下面顺序做，不要一开始就追求复杂效果：

1. 先生成一个简单静态道具
2. 优先导出 `FBX`
3. 导入 Unity
4. 手动调整缩放和朝向
5. 检查贴图是否正常
6. 加碰撞体
7. 拖进你们的场景 `Assets/Scenes/1.unity` 或 `Assets/Scenes/2.unity` 里测试
8. 确认没问题后再继续批量生成别的道具

## 11. 最常见的坑

### 坑 1：AI 生成出来很好看，但不能直接商用级落地

AI 更像“快速出草稿资产”的工具，不是完全替代建模师。

### 坑 2：生成模型太复杂

如果提示词里没有强调低模、实时渲染、VR 使用，模型很可能过重。

### 坑 3：材质好看但导入后失真

这是常见现象，不一定是你操作错了，很多时候是格式、Shader、贴图通道不一致。

### 坑 4：一个模型能用，不代表一批模型都能直接用

建议先用一个模型打通流程，再批量做。

## 12. 我对当前项目的建议

结合当前项目已有资源情况，建议把 AI 生成模型先用于：

- 新增公园内小型垃圾道具
- 场景装饰物
- 交互拾取物
- 重复摆放的小物件

暂时不要把它作为以下内容的主方案：

- 主角色
- 复杂动画角色
- 核心建筑主体

## 13. 最终推荐结论

对当前项目，推荐你们这样做：

1. 先选一个 AI 3D 平台生成简单静态物体
2. 第一轮优先导出 `FBX`
3. 导入 Unity 做基础检查
4. 跑通后再尝试 `GLB + glTFast`
5. 等团队熟悉后，再补 Blender 做简单修模

这是目前最适合小白团队、成本最低、出结果最快的路径。
