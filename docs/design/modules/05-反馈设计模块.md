# 反馈设计模块

## 1. 模块定位

反馈设计模块负责用户完成操作后系统如何回应，包括正确反馈、错误反馈、进度反馈、结算反馈和环保影响反馈。它决定用户是否知道自己做对了什么、错在哪里、下一步该怎么做。

本项目的反馈设计应避免单纯惩罚错误，而是把错误变成学习机会。反馈要即时、清楚、温和，并能强化用户对分类规则的记忆。

## 2. 设计目标

- 让用户每次投放后立即获得明确结果。
- 用多感官反馈增强 VR 沉浸感。
- 对错误分类给出原因，而不是只显示错误。
- 通过得分、连击和复盘增强练习动力。
- 用环境变化表现环保行为的意义。

## 3. 反馈类型

| 类型 | 触发时机 | 表现方式 |
| --- | --- | --- |
| 正确反馈 | 垃圾投入正确桶 | 绿色光效、加分、积极音效、轻微震动 |
| 错误反馈 | 垃圾投入错误桶 | 温和提示音、文字解释、正确类别提示 |
| 进度反馈 | 每次分类后 | 当前完成数量、倒计时、连击数 |
| 结算反馈 | 本轮结束 | 得分、正确率、用时、错误清单 |
| 环保反馈 | 完成任务或高分时 | 场景变整洁、回收效率提升等可视化 |

## 4. 正确反馈设计

正确反馈应短促、积极、不遮挡视野。

推荐表现：

- 垃圾桶桶口出现绿色光效。
- 分数增加数字从桶口或 UI 区域轻微浮现。
- 播放轻快音效。
- 连续正确时显示连击提示。
- 物品进入桶内或淡出消失。

正确反馈的重点是强化行为，不需要长篇解释。对于教学关，可以额外显示一句知识提示。

## 5. 错误反馈设计

错误反馈应温和、明确、可重新尝试。

推荐表现：

- 播放轻微提示音或手柄震动。
- 当前垃圾桶出现短暂黄色/红色提示。
- UI 显示“正确类别 + 原因”。
- 垃圾不销毁，允许用户再次投放。

示例文案：

- “污染纸巾不能回收，应投入其他垃圾。”
- “旧电池可能污染环境，应投入有害垃圾。”
- “剩饭属于易腐有机垃圾，应投入厨余垃圾。”

## 6. 结算反馈设计

结算页应展示：

- 本轮得分。
- 正确率。
- 完成用时。
- 连续正确最高次数。
- 分错的垃圾列表。
- 每个错误对应的正确分类和原因。
- 下一步按钮：再来一次、进入下一关、返回主界面。

结算页应简洁，不堆叠过多文字。复盘内容可以分条显示，优先展示用户实际分错的物品。

## 7. 环保影响反馈

环保影响反馈用于连接游戏行为和可持续价值。

可采用以下方式：

- 分类正确后，场景垃圾减少，地面变干净。
- 完成本轮后，显示“本轮减少了 X 次错误分类”。
- 高正确率时，社区环境出现更明亮、整洁的状态。
- 通过图标展示“回收资源提升”“污染风险降低”。

文案应避免夸大真实环境效益，采用模拟表达，例如“本轮模拟提升了资源回收效率”。

## 8. 混元建模与视觉资产描述

反馈模块中部分内容可以用 Unity 特效实现，也可以提前准备低多边形实体资产或 UI 面板资产。发送给混元时，应重点描述“可视化符号”和“环境变化物体”，不要把所有反馈都做成复杂模型。

### 8.1 正确反馈资产

| 资产 | 描述 | 用途 |
| --- | --- | --- |
| 绿色光效环 | 一个半透明绿色圆环或发光环，放在垃圾桶桶口位置 | 正确投放瞬间闪烁 |
| 加分图标 | 简洁的 `+10` 或星形奖励符号，可做成平面 3D 字牌 | 表现得分增加 |
| 连击徽章 | 小型圆形徽章，带火花或数字区域 | 连续正确时显示 |

### 8.2 错误反馈资产

| 资产 | 描述 | 用途 |
| --- | --- | --- |
| 黄色提示环 | 半透明黄色或橙色圆环，短暂出现在错误桶口 | 温和提示错误 |
| 错误说明牌 | 悬浮小面板，圆角矩形，留出文字区域 | 显示“正确类别 + 原因” |
| 重试箭头 | 简洁箭头指向正确垃圾桶，可用半透明材质 | 引导用户重新投放 |

### 8.3 环保影响资产

| 资产 | 描述 | 用途 |
| --- | --- | --- |
| 清洁度变化地面 | 同一场景的两种状态：有少量垃圾/干净整洁 | 表现任务前后变化 |
| 回收资源图标 | 塑料瓶、纸张、金属罐被回收的简化图标 | 结算时展示环保意义 |
| 污染降低图标 | 盾牌、叶片、警示减少等符号 | 表现有害垃圾正确处理 |
| 社区整洁标识 | 小型公告牌或环保标语牌 | 完成任务后的正向反馈 |

### 8.4 视觉风格要求

- 所有反馈资产应明亮、简洁、符号化，不能遮挡主要操作。
- 正确反馈以绿色为主，错误反馈以黄色/橙色为主，避免过度使用强烈红色。
- 面板资产应预留文字区域，具体文字建议在 Unity 中用 UI 添加。
- 特效类资产不需要高面数，优先保证 VR 中清楚、轻量、不会掉帧。

## 9. 与其他模块的关系

| 关联模块 | 对接内容 |
| --- | --- |
| 分类规则 | 接收正确/错误判断结果 |
| 游戏机制 | 影响得分、连击、成就 |
| UI 与引导 | 展示提示、结算和复盘 |
| 动画声音 | 反馈通过光效、音效、震动实现 |
| 数据评估 | 反馈内容来自用户行为记录 |
| 场景设计 | 场景状态可随反馈变化 |

## 10. 验收标准

- 用户投放后能立刻知道结果。
- 错误反馈能解释原因。
- 反馈不会遮挡用户操作视线。
- 正确和错误反馈在视觉、声音上明显不同。
- 结算页能帮助用户复盘错误。
- 环保反馈能体现项目主题，而不只是显示分数。
