# 数据与评估模块

## 1. 模块定位

数据与评估模块负责记录用户在 VR 垃圾分类游戏中的行为表现，并为结算复盘、数据分析和答辩展示提供依据。它回答的问题是：用户是否学会了垃圾分类？哪些垃圾最容易分错？VR 体验是否比传统教学更有效？

首个 Demo 可以先做本地记录和结算展示，后续再扩展为更完整的数据分析系统。

## 2. 设计目标

- 记录用户每次分类行为。
- 支持结算页显示正确率、用时和错误复盘。
- 找出高频错误物品和易混淆类别。
- 为数据分析员提供可量化指标。
- 支持后续 A/B Testing 和学习效果评估。

## 3. 记录指标

| 指标 | 含义 | 用途 |
| --- | --- | --- |
| 总分类数量 | 本轮处理了多少垃圾 | 判断任务完成度 |
| 正确数量 | 投放正确的垃圾数量 | 计算正确率 |
| 错误数量 | 投放错误的垃圾数量 | 评估学习难点 |
| 正确率 | 正确数量 / 总数量 | 核心学习指标 |
| 总用时 | 完成本轮花费时间 | 衡量效率 |
| 单件判断用时 | 从抓取到投放的时间 | 找出犹豫物品 |
| 错误物品 | 哪些垃圾被分错 | 生成复盘内容 |
| 错误类别 | 投到了哪个错误垃圾桶 | 分析误区 |
| 重试次数 | 投错后重新尝试次数 | 衡量反馈有效性 |
| 连击次数 | 连续正确数量 | 衡量稳定掌握程度 |

## 4. 行为日志结构

每次投放可记录一条日志：

| 字段 | 示例 |
| --- | --- |
| RoundId | 001 |
| SceneName | 校园食堂 |
| GarbageName | 污染纸巾 |
| CorrectType | 其他垃圾 |
| SelectedBinType | 可回收物 |
| IsCorrect | false |
| TimeUsed | 6.4 秒 |
| RetryCount | 1 |
| Timestamp | 本地时间 |

首个 Demo 可先把数据保存在内存中，用于当轮结算；后续可扩展为本地 JSON 或 CSV。

## 5. 结算评估

结算页应基于数据生成：

- 本轮正确率。
- 本轮用时。
- 本轮分数。
- 最容易分错的垃圾。
- 错误分类原因。
- 是否建议进入复盘关。

例如：

> 本轮正确率 83%。你把“污染纸巾”投到了可回收物，但它被污染后不能回收，应投入其他垃圾。

## 6. 学习效果评估

若用于课程答辩或实验展示，可设计简单对比：

### 6.1 VR 组

用户通过 VR 游戏进行垃圾分类训练。

### 6.2 对照组

用户通过图文说明或视频学习垃圾分类。

### 6.3 对比指标

- 即时分类准确率。
- 两周后记忆留存率。
- 易混淆垃圾错误率。
- 用户参与度评分。
- 用户环保行动意愿问卷。

这样可以说明 VR 不只是好玩，还可能提升记忆和行为转化。

## 7. 数据可视化建议

答辩 PPT 可展示：

- VR 组与传统教学组正确率柱状图。
- 易混淆垃圾错误率排行。
- 用户完成时间折线图。
- 正确率随关卡变化趋势。
- 复盘前后错误率对比。

图表应简洁，重点突出“VR 训练提升分类准确率和记忆留存”的主题。

## 8. 混元建模与数据展示资产描述

数据与评估模块通常不需要复杂 3D 模型，但可以为结算页、数据面板和答辩展示准备统一风格的可视化资产。

### 8.1 数据展示面板

建模描述：

- 一个悬浮式数据看板，浅色背景，分为正确率、用时、错误数量、最高连击四个区域。
- 每个区域有一个清晰图标，例如对勾、计时器、警示符号、连击星星。
- 文字和数字建议由 Unity UI 或 PPT 后期添加，模型只提供面板和图标底板。

### 8.2 图表资产

| 资产 | 描述 | 用途 |
| --- | --- | --- |
| 柱状图底板 | 简洁坐标轴和 2-4 个柱体，可替换高度 | 展示 VR 组与传统组准确率 |
| 折线图底板 | 简洁坐标轴和折线点位 | 展示错误率随训练下降 |
| 环形进度图 | 圆环进度条造型 | 展示正确率 |
| 错误排行面板 | 3-5 个条目列表，带小垃圾图标位置 | 展示易错垃圾 |

### 8.3 错误复盘可视化资产

建模描述：

- 一个“错误物品展示板”，左侧放用户分错的垃圾模型，右侧分为“误投类别”和“正确类别”两栏。
- 两栏之间可用箭头连接，表示从错误理解到正确知识的转化。
- 风格应像学习复盘，不像惩罚警告。

### 8.4 答辩展示建议

- 数据面板资产可以在 VR 结算页使用，也可以截图后放入 PPT。
- 图表视觉应保持环保主题，可用蓝、绿、红、灰四色对应四类垃圾。
- 不建议建复杂动态仪表盘，首版以清晰静态面板为主。

## 9. 隐私与边界

- Demo 阶段只记录游戏行为，不记录敏感个人信息。
- 若用于真实用户测试，需要说明数据用途。
- 数据结论应保持中立，不夸大 VR 的环保效果。
- 环保影响数据如果是模拟值，应明确为模拟展示。

## 10. 与其他模块的关系

| 关联模块 | 对接内容 |
| --- | --- |
| 游戏机制 | 提供得分、正确率、连击统计 |
| 反馈设计 | 错误记录生成复盘反馈 |
| 分类规则 | 记录正确类别和错误类别 |
| UI 与引导 | 在结算页展示数据 |
| 垃圾物品设计 | 分析哪些垃圾最容易分错 |
| 答辩展示 | 输出图表和量化依据 |

## 11. 验收标准

- 每次投放都能记录正确或错误。
- 结算页能显示正确率、用时和错误物品。
- 错误记录能对应到具体垃圾和错误垃圾桶。
- 数据能支持复盘关或复盘说明。
- 指标足够支撑答辩中的“VR 教育效果”分析。
- 不记录与项目无关的个人隐私信息。
