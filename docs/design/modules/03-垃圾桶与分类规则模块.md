# 垃圾桶与分类规则模块

## 1. 模块定位

垃圾桶与分类规则模块负责定义四类垃圾桶的视觉标识、空间摆放、投放检测区域和分类判断逻辑。它连接垃圾物品和用户投放行为，是系统判断“投对还是投错”的核心模块。

该模块必须同时服务用户理解和程序实现：用户需要一眼看懂垃圾桶类别，系统需要准确读取垃圾属性和垃圾桶类型并完成判断。

## 2. 设计目标

- 建立清晰统一的四分类垃圾桶视觉规范。
- 保证用户能通过颜色、图标、文字识别垃圾桶。
- 提供稳定、宽容的投放检测区域。
- 支持基于数据的分类规则配置。
- 为错误反馈提供准确解释。

## 3. 四分类垃圾桶规范

| 垃圾桶 | 主色 | 图标建议 | 标签文字 | 设计重点 |
| --- | --- | --- | --- | --- |
| 可回收物 | 蓝色 | 循环箭头 | 可回收物 | 强调资源再利用 |
| 有害垃圾 | 红色 | 警示标识 | 有害垃圾 | 强调危险与特殊处理 |
| 厨余垃圾 | 绿色 | 叶片/食物 | 厨余垃圾 | 强调易腐有机物 |
| 其他垃圾 | 灰色/黑色 | 普通垃圾图标 | 其他垃圾 | 表示无法归入前三类 |

垃圾桶不能只依赖颜色，应同时使用文字和图标，方便色弱用户和新手识别。

## 4. 分类规则

### 4.1 基础规则

每个垃圾物品拥有一个标准分类。用户将物品投入对应垃圾桶时，系统判断为正确；投入其他垃圾桶时，系统判断为错误，并给出解释。

示例：

| 物品 | 正确类别 | 错误说明示例 |
| --- | --- | --- |
| 旧电池 | 有害垃圾 | 电池可能含有污染物质，需要投入有害垃圾桶 |
| 剩饭 | 厨余垃圾 | 剩饭属于易腐有机垃圾，应投入厨余垃圾桶 |
| 塑料瓶 | 可回收物 | 清洁塑料瓶可以回收，应投入可回收物桶 |
| 污染纸巾 | 其他垃圾 | 污染纸巾不可回收，应投入其他垃圾桶 |

### 4.2 易混淆规则

部分垃圾需要额外解释，避免用户只记住答案而不理解原因。

- 奶茶杯：如果包含残留液体、吸管和杯盖，可作为混合垃圾教学，Demo 中建议统一归入其他垃圾。
- 外卖盒：干净纸盒可回收，明显油污餐盒归入其他垃圾。
- 纸张：干净废纸可回收，严重污染纸张归入其他垃圾。
- 药品：即使外包装类似纸盒，过期药品本体仍属于有害垃圾。

## 5. 投放检测设计

每个垃圾桶应设置一个桶口触发区域：

1. 垃圾物体进入触发区域。
2. 系统读取垃圾物品分类。
3. 系统读取当前垃圾桶类型。
4. 比对二者是否一致。
5. 触发正确或错误反馈。

检测区域应比桶口略大，避免玩家已经完成投放动作却因精度不足被判失败。

## 6. 规则数据结构建议

| 字段 | 含义 |
| --- | --- |
| BinType | 垃圾桶类型 |
| GarbageType | 垃圾物品类别 |
| IsCorrect | 投放是否匹配 |
| CorrectMessage | 正确时提示 |
| ErrorMessage | 错误时解释 |
| AllowRetry | 是否允许重新尝试 |

## 7. 混元建模资产描述

四分类垃圾桶是项目中最重要的实体资产之一，需要同时满足“现实可识别”和“VR 交互清晰”。建议四个垃圾桶使用统一造型，只改变颜色、图标和文字，形成完整系列。

### 7.1 通用垃圾桶造型

建模描述：

- 一组四个并排摆放的分类垃圾桶，造型现代、简洁、略带卡通低多边形风格。
- 每个垃圾桶高度约到成人腰部，顶部有清晰桶口，桶身正面有大面积标签区域。
- 桶盖可以是轻微上翻或带投放口的结构，方便玩家理解“从这里投进去”。
- 桶身边缘圆润，颜色饱和但不刺眼。

建模注意：

- 四个垃圾桶最好单独导出，也可以同时提供一组组合模型。
- 桶口位置应明显，便于 Unity 添加触发检测区域。
- 标签图标和文字可以建模为平面贴图区域，后续在 Unity 中替换。

### 7.2 单个垃圾桶描述

| 垃圾桶 | 建模描述 | 标签设计 |
| --- | --- | --- |
| 可回收物桶 | 蓝色桶身，正面有循环箭头图标，桶口可以略宽，体现可回收投放 | 文字“可回收物”，图标为三角循环箭头 |
| 有害垃圾桶 | 红色桶身，正面有警示三角或感叹号图标，视觉上更醒目 | 文字“有害垃圾”，图标为警示符号 |
| 厨余垃圾桶 | 绿色桶身，正面有叶片、果皮或食物图标 | 文字“厨余垃圾”，图标为叶片/餐厨图案 |
| 其他垃圾桶 | 灰色或黑色桶身，正面有普通垃圾图标 | 文字“其他垃圾”，图标为普通垃圾袋或垃圾桶 |

### 7.3 投放检测区域建模提示

- 桶口应有明显空间范围，不要做成完全封闭的平面。
- 桶口上方可以留出一个透明触发区域，Unity 中用于检测垃圾是否进入。
- 桶盖动画如果暂时不实现，模型也应看起来能投放。
- 四个桶之间保持一定间距，避免玩家投放时误触相邻桶。

### 7.4 可选辅助资产

- 垃圾分类说明牌：立在垃圾桶后方或旁边，用四种颜色展示分类规则。
- 桶口高亮环：用于正确/错误反馈，可建模为薄环或在 Unity 中用特效实现。
- 地面定位贴纸：四个桶前方的颜色区域，帮助玩家快速识别投放位置。

## 8. 与其他模块的关系

| 关联模块 | 对接内容 |
| --- | --- |
| 垃圾物品设计 | 读取每个物品的分类属性 |
| 交互设计 | 接收用户投放动作 |
| 反馈设计 | 提供正确/错误结果 |
| 游戏机制 | 影响得分、连击和关卡状态 |
| UI 与引导 | 展示桶标签、解释文本 |
| 数据评估 | 记录错误投放类型 |

## 9. 验收标准

- 四类垃圾桶在颜色、文字、图标上清晰区分。
- 用户不看说明也能大致理解垃圾桶类型。
- 投放检测稳定，不频繁误判。
- 每个垃圾都有唯一明确的默认分类。
- 错误提示能解释原因，而不是只显示“错误”。
- 规则配置便于新增垃圾和新增关卡。
