# VR 垃圾分类小游戏模块设计文档目录

本目录将 ParkClean VR 的总体设计拆分为 8 个可独立推进的模块。每个模块都可以单独用于分工、答辩展开、后续开发对接，也可以作为发送给混元等 AI 建模工具的需求参考。

| 序号 | 模块 | 文档 | 主要职责 |
| --- | --- | --- | --- |
| 01 | 场景设计 | [01-场景设计模块.md](01-场景设计模块.md) | 定义校园、宿舍、社区等 VR 环境与空间布局 |
| 02 | 垃圾物品设计 | [02-垃圾物品设计模块.md](02-垃圾物品设计模块.md) | 定义四类垃圾、易混淆物品与解释文案 |
| 03 | 垃圾桶与分类规则 | [03-垃圾桶与分类规则模块.md](03-垃圾桶与分类规则模块.md) | 定义垃圾桶标识、投放检测和分类判断 |
| 04 | 交互设计 | [04-交互设计模块.md](04-交互设计模块.md) | 定义观察、抓取、移动、投放和重试流程 |
| 05 | 反馈设计 | [05-反馈设计模块.md](05-反馈设计模块.md) | 定义正确、错误、结算和环保影响反馈 |
| 06 | 游戏机制 | [06-游戏机制模块.md](06-游戏机制模块.md) | 定义任务、倒计时、得分、关卡、成就和复盘 |
| 07 | UI 与引导 | [07-UI与引导模块.md](07-UI与引导模块.md) | 定义任务提示、新手引导、过程 UI 和结算页 |
| 08 | 数据与评估 | [08-数据与评估模块.md](08-数据与评估模块.md) | 定义行为记录、正确率、错误分析和答辩图表 |

推荐阅读顺序：

1. 先读总体设计文档：[../VR垃圾分类小游戏总体设计文档.md](../VR垃圾分类小游戏总体设计文档.md)
2. 再读本目录中的 8 个模块文档。
3. 若用于答辩，设计师部分可重点展开 01-06；数据分析员部分重点使用 08；开发工程师部分重点对接 03、04、06、08。

建模使用建议：

1. 需要生成场景时，优先查看 `01-场景设计模块.md` 的“混元建模资产描述”。
2. 需要生成单个垃圾物品时，优先查看 `02-垃圾物品设计模块.md` 的“混元建模资产描述”。
3. 需要生成垃圾桶、虚拟手、UI 面板、成就徽章、数据看板等实体时，查看对应模块中的“混元建模”章节。
4. 发送给混元时建议一次只发送一个资产或一组同类资产，避免一次生成过多导致细节混乱。
