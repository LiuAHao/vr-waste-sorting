# UI 与引导模块

## 1. 模块定位

UI 与引导模块负责游戏中的界面信息和新手教学，包括开始界面、任务提示、倒计时、进度、分类反馈、结算页和操作引导。它帮助用户理解任务、掌握操作、查看结果。

VR 中的 UI 不能照搬普通屏幕界面。文字要少、字号要大、位置要舒适，重要信息要能在用户自然视野中被看到。

## 2. 设计目标

- 让用户快速理解当前任务。
- 帮助初次 VR 用户学会抓取和投放。
- 在不遮挡视线的前提下展示必要信息。
- 用简洁文案解释分类错误。
- 支持结算复盘和下一步操作。

## 3. UI 信息层级

| 层级 | 内容 | 设计要求 |
| --- | --- | --- |
| 高优先级 | 倒计时、任务进度、正确/错误反馈 | 始终清晰、短文本 |
| 中优先级 | 当前分数、连击、任务说明 | 可固定在视野边缘 |
| 低优先级 | 成就、环保影响、详细复盘 | 结算页集中展示 |

核心原则：游戏过程中少显示文字，结算和复盘时再展示更多信息。

## 4. 新手引导

教学关建议包含以下步骤：

1. 看向第一个高亮垃圾。
2. 按下抓取键拿起垃圾。
3. 看向对应垃圾桶。
4. 将垃圾放入桶口。
5. 查看正确反馈。
6. 自主完成下一件垃圾。

引导方式可包括：

- 箭头指示。
- 高亮描边。
- 简短文字。
- 语音提示。
- 垃圾桶闪烁提示。

## 5. 游戏中 UI

### 5.1 常驻信息

- 倒计时。
- 当前目标进度，例如“清扫目标：4/12”或“分类进度：4/12”。
- 当前分数。
- 当前连击数。

常驻 UI 应固定在舒适视野范围，例如前方上方或手腕面板，不应贴脸。

### 5.2 临时提示

- “分类正确”
- “请投入厨余垃圾桶”
- “污染纸巾属于其他垃圾”
- “时间剩余 30 秒”

临时提示出现时间应短，避免持续遮挡。

## 6. 结算页设计

结算页建议包含：

- 结果标题：任务完成/挑战失败。
- 得分。
- 正确率。
- 用时。
- 错误复盘列表。
- 环保影响提示。
- 操作按钮：重玩、下一关、返回。

错误复盘列表应优先展示最关键的 3-5 个错误，避免信息过载。

## 7. 文案原则

- 一句话只表达一个信息。
- 错误解释控制在 20 字左右。
- 避免专业术语堆叠。
- 用“应该投入……”代替“你错了”。
- 鼓励性文案要适度，不要过度游戏化。

示例：

- “旧电池属于有害垃圾。”
- “剩饭是厨余垃圾。”
- “污染纸巾不能回收，应投其他垃圾。”

## 8. 混元建模与 UI 资产描述

UI 与引导模块中，部分界面可以由 Unity UI 实现，但仍可让混元生成统一风格的 3D 面板、箭头、按钮底板和提示牌。所有 UI 资产应保证 VR 中清晰可读，文字内容建议后期在 Unity 中添加，不直接建死在模型上。

### 8.1 任务提示面板

建模描述：

- 一个悬浮式浅色半透明面板，圆角矩形，边框简洁。
- 面板顶部有小图标区域，可放垃圾桶或任务图标。
- 中间预留文字区域，用于显示“请在 180 秒内完成 12 件垃圾分类”。
- 面板尺寸适合放在玩家前方 1.5-2 米处阅读。

### 8.2 倒计时与进度面板

建模描述：

- 小型悬浮信息条，包含倒计时区域、进度区域和分数区域。
- 造型简洁，不要像复杂仪表盘。
- 可设计为手腕面板或前方上方固定面板。

建议元素：

- 时钟图标。
- 垃圾袋或垃圾桶进度图标。
- 星星或积分图标。

### 8.3 新手引导资产

| 资产 | 建模描述 | 用途 |
| --- | --- | --- |
| 引导箭头 | 半透明蓝色或绿色箭头，方向明确，边缘柔和 | 指向垃圾或垃圾桶 |
| 高亮光圈 | 圆形或椭圆形光圈，放在垃圾底部 | 标出可交互目标 |
| 操作提示牌 | 小型悬浮牌，预留文字区域 | 显示“按键抓取”等提示 |
| 视线引导点 | 小圆点或发光点 | 引导用户看向目标 |

### 8.4 结算页面板

建模描述：

- 一个较大的悬浮结算面板，适合玩家站立或坐姿观看。
- 面板分为标题区、数据区、错误复盘区、按钮区。
- 错误复盘区预留 3-5 个列表条目，每个条目可配一个垃圾小图标。
- 按钮底板包含“重玩”“下一关”“返回”三个样式。

### 8.5 视觉规范

- UI 面板颜色以浅色、低透明度背景为主，文字区域对比明确。
- 图标线条粗一些，避免 VR 中看不清。
- 不把中文文字直接做进模型，避免后续修改困难。
- 面板边缘不使用强反光材质。
- 所有 UI 资产应保持同一风格，避免像不同素材包拼接。

## 9. 与其他模块的关系

| 关联模块 | 对接内容 |
| --- | --- |
| 交互设计 | 展示抓取、投放、重试提示 |
| 反馈设计 | 承载正确/错误信息 |
| 游戏机制 | 展示倒计时、分数、进度、结算 |
| 分类规则 | 展示正确类别和原因 |
| 数据评估 | 结算页读取用户表现数据 |
| 无障碍需求 | 支持大字号、语音和图标辅助 |

## 10. 验收标准

- 用户能通过 UI 明确当前任务。
- 引导能让新手完成第一件垃圾分类。
- 游戏中 UI 不遮挡主要操作区域。
- 错误解释简短清楚。
- 结算页能展示分数、正确率、用时和错误复盘。
- UI 适合 VR 阅读，不出现过小文字。
