# 场景设计模块

## 1. 模块定位

场景设计模块负责定义玩家进入 VR 后所处的虚拟环境，包括空间主题、垃圾分布、垃圾桶位置、动线范围、环境氛围和任务发生方式。它决定用户是否能自然理解“我在哪里、我要做什么、这些垃圾为什么会出现在这里”。

本项目的场景不追求宏大开放世界，而是选择贴近生活、范围可控、便于投放操作的环保教育场景。场景应服务垃圾分类学习，让玩家感觉自己正在完成一个真实生活中也会遇到的分类任务。

## 2. 设计目标

- 建立生活化、可理解的 VR 垃圾分类空间。
- 让用户进入场景后快速明确任务范围。
- 控制移动距离和视角变化，降低 VR 眩晕。
- 为垃圾物品、垃圾桶和反馈系统提供清晰的空间关系。
- 支持后续扩展多个主题场景。

## 3. 场景类型

| 场景 | 主要垃圾 | 教学重点 | 适合阶段 |
| --- | --- | --- | --- |
| 校园食堂 | 剩饭、果皮、餐盒、纸巾、饮料瓶 | 厨余垃圾与其他垃圾区分 | 基础关 |
| 学生宿舍 | 快递纸箱、塑料袋、旧衣物、电池、包装盒 | 可回收物和有害垃圾识别 | 基础关/挑战关 |
| 社区投放点 | 玻璃瓶、过期药品、混合外卖垃圾 | 多类别综合判断 | 挑战关 |
| 公共空间 | 饮料杯、易拉罐、包装纸、烟头 | 快速识别和投放 | 计时关 |

首个 Demo 建议优先选择“校园食堂”或“社区投放点”。校园食堂更贴近学生汇报语境，社区投放点更能体现垃圾分类主题。

## 4. 空间布局

### 4.1 基础布局

场景由三部分组成：

- 玩家起始区：用户进入 VR 后的位置，视野内应能看到主要任务区域。
- 垃圾分布区：桌面、地面或垃圾暂存区，用于摆放待分类物品。
- 垃圾桶区：四分类垃圾桶集中摆放，颜色、文字和图标都应清晰。

### 4.2 推荐动线

用户不需要大范围行走。推荐设计为“站立式小范围操作”：

1. 用户站在场景中央或桌面前。
2. 左右或前方可看到待分类垃圾。
3. 四类垃圾桶位于用户正前方或半环形区域。
4. 用户通过转身、伸手、短距离传送完成分类。

这样既能模拟现实行为，又能避免长距离移动造成眩晕。

## 5. 场景交互要点

- 可交互垃圾应和纯装饰物有明显区别，例如描边、高亮或摆放在任务区域。
- 垃圾桶应放在稳定、固定的位置，避免用户寻找成本过高。
- 场景中不应堆放过多动态物体，防止干扰判断和影响性能。
- 环境装饰应强化主题，例如食堂餐桌、宿舍书桌、社区公告牌。
- 任务开始时可通过箭头或高亮引导用户看向垃圾和垃圾桶。

## 6. 视觉氛围

整体风格建议采用明亮、干净、半写实或低多边形风格。环保教育场景不适合过度阴暗、压抑或复杂。场景可以在任务前略显凌乱，任务完成后逐渐变得整洁，用环境变化表现“分类行为带来的正向影响”。

## 7. 混元建模资产描述

本模块中的场景资产可发送给混元进行建模。建议统一使用“低多边形、明亮、干净、适合 Unity VR 实时运行”的风格，避免过多细碎装饰和高面数物体。所有场景应预留玩家站立点、垃圾摆放点和四分类垃圾桶摆放区。

### 7.1 校园食堂场景

建模描述：

- 一个现代校园食堂局部场景，包含 2-3 张浅色餐桌、长凳或椅子、地面瓷砖、背景窗口、墙面菜单牌和简单照明。
- 餐桌上有餐盘、碗、筷子、饮料瓶、纸巾、外卖餐盒等生活化物品，但待分类垃圾应集中放在玩家前方 1-2 米范围内。
- 视觉氛围明亮、干净、校园感强，不要做成脏乱差环境；只需要有少量散落垃圾来提示任务。
- 场景中央或桌边预留玩家站立区域，前方或右侧预留四个垃圾桶的位置。

建模注意：

- 桌椅、餐盘、背景装饰可以作为静态环境模型。
- 待分类垃圾不要和场景装饰合并成一个模型，应单独建模，方便 Unity 中抓取。
- 地面和墙面材质简洁，减少反光，避免 VR 中刺眼。

### 7.2 学生宿舍场景

建模描述：

- 一个大学生宿舍局部空间，包含书桌、床铺一角、柜子、地面、快递纸箱、书本、电脑或台灯等元素。
- 垃圾主要来自生活和快递场景，例如纸箱、塑料包装袋、废纸、电池、饮料瓶。
- 场景应体现“学生生活感”，但保持整洁，避免过度杂乱导致玩家找不到可交互垃圾。

建模注意：

- 快递纸箱、塑料瓶、电池等任务物品必须单独拆分。
- 背景物品可以低精度处理，重点保证待分类物品清晰。

### 7.3 社区垃圾投放点场景

建模描述：

- 一个社区户外垃圾分类投放点，包含地面、墙面或围栏、环保宣传栏、四分类垃圾桶、简化绿植和小区背景。
- 场景中心是四个分类垃圾桶，旁边有少量待分类垃圾暂放区。
- 适合表现正式垃圾分类教学，视觉上要清楚、干净、有公共设施感。

建模注意：

- 四分类垃圾桶建议单独建模，方便替换颜色和标签。
- 宣传栏可做成低多边形平面，文字在 Unity 中后期贴图或 UI 添加。

### 7.4 公共空间场景

建模描述：

- 一个小型公共休息区或校园广场角落，包含长椅、地砖、路灯、公告牌、简化树木和垃圾桶区域。
- 垃圾包括饮料杯、易拉罐、包装纸、烟头等轻量物品，适合计时挑战。
- 场景应开阔但不大，玩家站在中心即可完成主要操作。

建模注意：

- 树木、路灯和长椅作为环境氛围物，不要影响玩家视线。
- 小物品如烟头不建议做得过小，可做成略放大的可抓取物或放在小托盘上。

### 7.5 通用场景导出要求

- 场景风格：低多边形或半写实卡通，明亮、亲和、适合环保教育。
- 模型结构：环境物体与可交互垃圾分开导出。
- 尺寸比例：接近真实生活比例，保证桌面高度、垃圾桶高度适合 VR 站立操作。
- 材质要求：颜色清晰，贴图简洁，避免复杂透明和强反光。
- 性能要求：控制面数，适合 Unity VR 实时渲染。

## 8. 与其他模块的关系

| 关联模块 | 对接内容 |
| --- | --- |
| 垃圾物品设计 | 决定每个场景出现哪些垃圾 |
| 垃圾桶与分类规则 | 决定垃圾桶摆放、标识和分类入口 |
| 交互设计 | 决定用户移动、抓取和投放距离 |
| 反馈设计 | 场景是否根据正确分类变整洁 |
| 游戏机制 | 场景对应关卡难度和任务目标 |
| UI 与引导 | 场景内任务提示、箭头和高亮 |

## 9. 验收标准

- 用户进入场景后 10 秒内能理解自己处在什么环境。
- 用户能在第一视野范围内看到待分类垃圾或任务提示。
- 四类垃圾桶位置清晰，不需要长时间寻找。
- 场景范围适合 VR 操作，不依赖快速奔跑或大范围移动。
- 场景中的垃圾类型与场景主题一致。
- 场景资源数量可控，运行时保持稳定帧率。
