# 垃圾物品设计模块

## 1. 模块定位

垃圾物品设计模块负责定义游戏中出现的待分类物品，包括垃圾名称、所属类别、外观表现、易错点、解释文案和难度等级。它是垃圾分类学习内容的核心，直接决定玩家能学到什么。

垃圾物品不能只是简单模型或图标，而应尽量贴近日常生活。尤其是奶茶杯、外卖盒、污染纸巾这类现实中容易分错的垃圾，需要通过视觉细节和反馈解释体现学习价值。

## 2. 设计目标

- 覆盖四类垃圾的典型物品。
- 保证物品外观可识别、可抓取、可投放。
- 加入易混淆物品，提升教育意义。
- 为每个物品配置清晰的错误解释。
- 支持通过数据配置扩展新垃圾。

## 3. 垃圾分类清单

### 3.1 可回收物

| 物品 | 视觉特征 | 说明 |
| --- | --- | --- |
| 塑料瓶 | 透明瓶身、瓶盖 | 清洁塑料容器可回收 |
| 废纸 | 干净纸张、书页 | 未被污染的纸类可回收 |
| 纸箱 | 快递箱、纸板纹理 | 保持干燥时可回收 |
| 易拉罐 | 金属材质、拉环 | 金属容器可回收 |
| 玻璃瓶 | 透明或绿色玻璃 | 玻璃制品可回收 |

### 3.2 厨余垃圾

| 物品 | 视觉特征 | 说明 |
| --- | --- | --- |
| 剩饭 | 碗中米饭或饭粒 | 易腐有机垃圾 |
| 果皮 | 苹果皮、香蕉皮 | 来自食物处理过程 |
| 菜叶 | 绿色叶片 | 厨房有机废弃物 |
| 茶叶渣 | 小堆茶渣 | 易腐厨余 |

### 3.3 有害垃圾

| 物品 | 视觉特征 | 说明 |
| --- | --- | --- |
| 旧电池 | 电池外形、正负极 | 可能含有污染物质 |
| 过期药品 | 药盒、药瓶 | 需要特殊处理 |
| 灯管 | 细长玻璃管 | 可能含有有害成分 |
| 油漆桶 | 警示标识、残留涂料 | 化学残留有污染风险 |

### 3.4 其他垃圾

| 物品 | 视觉特征 | 说明 |
| --- | --- | --- |
| 污染纸巾 | 油污、水渍、皱褶 | 被污染后不可回收 |
| 奶茶杯 | 杯身、吸管、残留液体 | 常见混合垃圾，需作为易错教学 |
| 油污外卖盒 | 油渍明显 | 被食物污染后不宜回收 |
| 烟头 | 小型、灰黑色 | 不属于可回收物 |

## 4. 难度分级

| 难度 | 物品类型 | 设计目的 |
| --- | --- | --- |
| 初级 | 塑料瓶、果皮、电池、纸巾 | 建立四分类基本概念 |
| 中级 | 纸箱、玻璃瓶、剩饭、药品 | 加入更多日常物品 |
| 高级 | 奶茶杯、油污餐盒、污染纸张 | 训练易混淆判断 |

初级物品应一眼可辨，高级物品应包含材质、污染程度或残留物等判断线索。

## 5. 物品信息结构

每个垃圾物品建议配置以下信息：

| 字段 | 含义 |
| --- | --- |
| 名称 | 展示给用户的物品名 |
| 分类 | 可回收物、厨余垃圾、有害垃圾、其他垃圾 |
| 难度 | 初级、中级、高级 |
| 正确解释 | 投对后可展示的简短知识 |
| 错误解释 | 投错后解释为什么不属于该桶 |
| 场景来源 | 食堂、宿舍、社区、公共空间 |
| 模型资源 | 对应 Unity 模型或预制体 |

## 6. 视觉设计原则

- 物品尺寸要适合 VR 抓取，不宜过小。
- 重要判断线索应直接可见，例如油污、残留液体、危险标识。
- 同类物品可用统一风格，避免画风混乱。
- 易混淆垃圾要比普通垃圾有更多细节。
- 物体碰撞体应简化，保证性能和抓取稳定。

## 7. 混元建模资产描述

以下内容可作为发送给混元的建模需求。所有垃圾物品统一采用“低多边形或半写实卡通风格，适合 Unity VR 抓取交互，单独模型导出，材质清晰，轮廓易识别”。每个物品的尺寸可略大于真实比例，方便 VR 玩家抓取和观察。

### 7.1 通用建模要求

- 模型用途：VR 垃圾分类小游戏中的可抓取、可投放物品。
- 风格：明亮、干净、半写实卡通或低多边形，不要恐怖、脏污过度或写实到令人不适。
- 尺寸：以玩家手柄可抓取为标准，小物体可以适当放大。
- 结构：每个垃圾单独导出，不要和桌面、地面、垃圾桶合并。
- 细节：分类判断线索必须可见，例如油污、残留液体、危险标识、材质纹理。
- 碰撞：形状尽量简洁，便于 Unity 添加 Box Collider、Capsule Collider 或 Mesh Collider。
- 贴图：不使用真实品牌 Logo，可使用无品牌简化图案。
- 命名：建议使用英文或拼音资产名，例如 `Garbage_PlasticBottle`、`Garbage_Battery`。

### 7.2 可回收物建模说明

| 物品 | 建模描述 | 关键细节 | VR 注意 |
| --- | --- | --- | --- |
| 塑料瓶 | 透明或半透明矿泉水瓶，瓶身略有压痕，带浅蓝色瓶盖，瓶身可有简化标签但不要真实品牌 | 透明材质、瓶盖、圆柱瓶身、轻微塑料反光 | 形状细长，建议加粗一点，方便抓取 |
| 废纸 | 几张干净白纸或浅色纸张叠在一起，边缘略微弯曲 | 干净、无污渍、纸张平整，与污染纸巾形成对比 | 单张纸太薄不好抓，可做成 3-5 张纸的小叠 |
| 纸箱 | 小型快递纸箱，棕色纸板材质，带胶带和折痕 | 纸板纹理、封箱胶带、箱体边缘 | 做成手持大小，不要过大 |
| 易拉罐 | 银色或彩色金属饮料罐，顶部有拉环，罐身可轻微压扁 | 金属材质、拉环、圆柱形、轻微凹陷 | 适合单手抓取，碰撞体用圆柱近似 |
| 玻璃瓶 | 透明或绿色玻璃瓶，瓶口较窄，瓶身有厚玻璃感 | 玻璃材质、瓶口、瓶底厚度 | 透明度不要太高，否则 VR 中不易看清 |

### 7.3 厨余垃圾建模说明

| 物品 | 建模描述 | 关键细节 | VR 注意 |
| --- | --- | --- | --- |
| 剩饭 | 一个小碗或餐盘中的米饭残留，可带少量菜叶和饭粒 | 米饭颗粒、餐盘、食物残渣 | 不建议做散落单粒米饭，应做成盘装整体 |
| 果皮 | 香蕉皮或苹果皮，小堆放置，颜色鲜明 | 香蕉皮弯曲形状或苹果皮螺旋形，带自然颜色 | 果皮可做成一小堆，避免太薄不好抓 |
| 菜叶 | 几片绿色菜叶堆在一起，边缘略卷曲 | 绿色叶片、叶脉、柔软感 | 做成可抓取的一小团，不要过于扁平 |
| 茶叶渣 | 小堆深绿色或褐色茶叶渣，放在小碟或滤网中 | 颗粒感、湿润感、小碟承载 | 茶叶渣本体太碎，建议和小碟作为一个可抓取整体 |

### 7.4 有害垃圾建模说明

| 物品 | 建模描述 | 关键细节 | VR 注意 |
| --- | --- | --- | --- |
| 旧电池 | 一节或两节圆柱电池，带正负极标识和警示色块 | 正负极、金属头、危险提示符号 | 可适当放大，避免太小难抓 |
| 过期药品 | 小药盒和药瓶组合，药盒上有简化十字或药品图标，避免真实药名 | 药盒、药瓶、过期标签、医疗符号 | 建议做成药盒+药瓶一组，识别度更高 |
| 灯管 | 短款废旧荧光灯管或节能灯，玻璃管加两端金属接口 | 细长玻璃管、金属端头、易碎感 | 太长会影响操作，建议缩短为可手持长度 |
| 油漆桶 | 小型油漆桶，桶身有警示标签，桶口可露出少量彩色残留涂料 | 金属桶、提手、警示标识、涂料残留 | 手持尺寸，桶身不要过大 |

### 7.5 其他垃圾建模说明

| 物品 | 建模描述 | 关键细节 | VR 注意 |
| --- | --- | --- | --- |
| 污染纸巾 | 一团皱起的纸巾，带明显油渍、水渍或污渍 | 褶皱、污渍、湿皱感，与干净废纸区分 | 做成纸团，便于抓取 |
| 奶茶杯 | 透明塑料杯或纸杯，带杯盖、吸管、残留液体和少量珍珠颗粒 | 杯盖、吸管、残留液体、混合材质 | Demo 中统一作为其他垃圾，杯身不使用真实品牌 |
| 油污外卖盒 | 白色或米色外卖餐盒，打开或半开，内部有油渍和食物残留 | 油污、餐盒盖、食物残渣 | 做成整体模型，避免盖子独立掉落 |
| 烟头 | 几个烟头放在小托盘或纸片上，烟灰和滤嘴清晰 | 灰黑烟灰、黄色滤嘴、小体积 | 烟头太小，建议做成“烟头小堆”整体 |

### 7.6 首批建模优先级

首批 Demo 建议先建 12 个物品，每类 3 个，保证范围可控：

| 类别 | 首批物品 |
| --- | --- |
| 可回收物 | 塑料瓶、纸箱、易拉罐 |
| 厨余垃圾 | 剩饭、果皮、菜叶 |
| 有害垃圾 | 旧电池、过期药品、灯管 |
| 其他垃圾 | 污染纸巾、奶茶杯、油污外卖盒 |

### 7.7 分类口径说明

现实垃圾分类规则会因地区和处理方式不同而存在差异。为了保证 Demo 规则清晰，本项目采用简化教学口径：

- 干净纸张、纸箱、塑料瓶、易拉罐、玻璃瓶归为可回收物。
- 剩饭、果皮、菜叶、茶叶渣归为厨余垃圾。
- 旧电池、过期药品、灯管、油漆桶归为有害垃圾。
- 污染纸巾、带残留液体的奶茶杯、油污外卖盒、烟头归为其他垃圾。

其中奶茶杯和外卖盒属于易混淆教学物品，模型中必须体现“残留液体”或“油污”，否则用户会误以为它们应当回收。

## 8. 与其他模块的关系

| 关联模块 | 对接内容 |
| --- | --- |
| 场景设计 | 决定物品出现在哪个生活场景 |
| 分类规则 | 每个物品对应正确垃圾类别 |
| 交互设计 | 物品需要能被抓取、移动、投放 |
| 反馈设计 | 正误解释来自物品配置 |
| 游戏机制 | 难度和得分与物品复杂度相关 |
| 数据评估 | 记录哪些物品最容易被分错 |

## 9. 验收标准

- 每类垃圾至少包含 2 个典型物品。
- 首个 Demo 至少包含 8-12 个可分类物品。
- 每个物品都有明确分类和解释文案。
- 易混淆物品具备可观察的判断线索。
- 物品模型尺寸适合 VR 抓取。
- 投放判断不依赖用户猜测，用户能通过观察做出选择。
