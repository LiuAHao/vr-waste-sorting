# 交互设计模块

## 1. 模块定位

交互设计模块负责定义玩家在 VR 中如何完成垃圾分类动作，包括观察、抓取、移动、判断、投放和重新尝试。它是项目区别于普通答题游戏的关键：玩家不是点击选项，而是在虚拟空间中亲手完成分类。

交互设计应尽量接近现实生活，同时降低 VR 操作门槛，让初次使用者也能顺利完成任务。

## 2. 设计目标

- 用身体动作强化垃圾分类记忆。
- 保证抓取、移动和投放过程自然顺畅。
- 减少复杂菜单和抽象按钮。
- 降低操作精度要求，避免挫败。
- 支持手柄交互，并为后续手势交互预留空间。

## 3. 核心交互流程

```mermaid
flowchart LR
    A[观察垃圾] --> B[伸手或射线选中]
    B --> C[抓取物品]
    C --> D[移动到垃圾桶]
    D --> E[投放到桶口]
    E --> F[系统判断]
    F --> G[反馈与下一件垃圾]
```

## 4. 交互动作设计

| 动作 | 用户行为 | 系统表现 |
| --- | --- | --- |
| 观察 | 看向垃圾物品 | 可交互垃圾出现轻微描边或标签 |
| 选中 | 手柄靠近或射线指向 | 物品高亮，提示可抓取 |
| 抓取 | 按下手柄抓取键 | 物品跟随手部移动 |
| 移动 | 手持物品转向垃圾桶 | 物体保持稳定，不抖动 |
| 投放 | 松开抓取键或放入桶口 | 触发桶口检测 |
| 重试 | 投错后重新拿起 | 物品回到可操作状态 |

## 5. 操作方式

### 5.1 近距离抓取

适用于桌面或地面附近的垃圾。玩家伸手靠近物品后按键抓取，适合强调真实感。

优点：沉浸感强，动作接近现实。  
限制：对玩家位置和物品摆放要求更高。

### 5.2 射线抓取

适用于不方便弯腰或距离稍远的物品。玩家用手柄射线指向物品后抓取。

优点：降低眩晕和弯腰负担。  
限制：真实感略弱。

### 5.3 推荐方案

首个 Demo 推荐支持射线抓取或近距离抓取中的一种稳定方案。若时间允许，可同时支持近距离抓取和射线辅助，确保不同用户都能上手。

## 6. 防眩晕交互原则

- 不设计高速奔跑。
- 不强制用户频繁转身超过 180 度。
- 垃圾和垃圾桶应位于舒适视野范围内。
- 支持坐姿或站姿完成任务。
- 抓取和投放动画要平滑，不要突然跳变。
- 传送点数量少而清晰，避免用户迷路。

## 7. 错误交互

投错时不应立即销毁物品。推荐流程：

1. 垃圾进入错误桶口。
2. 系统播放轻微错误提示。
3. 显示正确类别和简短原因。
4. 垃圾回到用户手中、桶口附近或原始位置。
5. 用户重新尝试。

这样能把错误转化为学习过程。

## 8. 混元建模与交互资产描述

交互模块中可能需要建模或制作的实体包括虚拟手、手柄射线、抓取提示、传送点和重试引导。它们不一定全部需要高精度模型，但需要在视觉上清楚、轻量，并适合 VR 中实时显示。

### 8.1 虚拟手模型

建模描述：

- 一双简洁的 VR 虚拟手，低多边形或半写实卡通风格，肤色中性，不强调具体性别和年龄。
- 手部姿态包括自然张开、握持、捏取三种基础状态。
- 手腕部分可以简化，不需要完整手臂。
- 材质柔和，不要过于写实，避免在 VR 中产生不适。

用途：

- 表示玩家手部位置。
- 抓取塑料瓶、纸箱、奶茶杯等物品时形成具身感。
- 配合动画表现“拿起”和“松开”。

### 8.2 手柄射线与选中提示

视觉描述：

- 一条半透明浅蓝色或白色射线，从手柄/手部向前延伸。
- 射线末端有小圆点或准星，指向可抓取垃圾时变亮。
- 被选中的垃圾出现浅色描边或底部光圈。

制作建议：

- 射线和描边可在 Unity 中实现，不一定发送混元建模。
- 如果需要模型资产，可制作“准星圆点”“选中光圈”“抓取图标”三个轻量平面模型。

### 8.3 抓取提示资产

| 资产 | 描述 | 用途 |
| --- | --- | --- |
| 抓取图标 | 简化手掌图标，带向内握合的线条 | 提示玩家可以抓取 |
| 投放图标 | 小垃圾桶加向下箭头 | 提示玩家把物品放入桶口 |
| 重试图标 | 回转箭头或重新投放箭头 | 投错后提示再次尝试 |
| 传送点 | 地面半透明圆环，中心有脚印或定位点 | 表示可安全移动位置 |

### 8.4 动作表现要求

- 抓取大型物品时手部呈满握姿态，例如塑料瓶、纸箱、奶茶杯。
- 抓取小型物品时手部呈捏取姿态，例如电池、药品盒。
- 投放时物品不要突然消失，应先进入桶口或触发短暂光效后再消失。
- 错误重试时，物品回到原位置或玩家手边，避免突然飞动造成眩晕。

## 9. 与其他模块的关系

| 关联模块 | 对接内容 |
| --- | --- |
| 场景设计 | 决定玩家移动范围和物品摆放 |
| 垃圾物品设计 | 决定物品尺寸、碰撞体和抓取方式 |
| 分类规则 | 投放后触发分类判断 |
| 反馈设计 | 接收判断结果并反馈给用户 |
| UI 与引导 | 提供抓取和投放提示 |
| 动画声音 | 匹配抓取、移动、投放动作 |

## 10. 验收标准

- 初次使用者能在短时间内完成抓取和投放。
- 抓取过程稳定，不出现明显抖动或穿模。
- 投放检测符合用户直觉。
- 投错后用户知道如何重新尝试。
- 不需要复杂菜单就能完成核心流程。
- 交互过程能体现“亲手分类”的 VR 价值。
