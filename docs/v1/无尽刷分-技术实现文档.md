# V1 无尽刷分技术实现文档

## 1. 玩法定位

无尽刷分模式是 V1 中重玩性最强的一条玩法线。它的核心不是固定目标，也不是标准关卡完成，而是让玩家在不断出现的垃圾中持续刷分，并随着游戏推进逐步提升难度。

该模式适合：

- 展示反复游玩的可能性
- 强调高分追求
- 做“最高分”和“最佳表现”演示

## 2. 核心规则

首版建议规则：

- 开局给定基础时间，例如 `180` 秒
- 场景始终保持一定数量垃圾
- 玩家处理垃圾后立即补新垃圾
- 随着时间推进，提高易混淆垃圾比例
- 时间结束后结算

如果首版想保守一点，无尽刷分可以先做成“时间内无限补货的高分模式”，不额外加入失败条件。

后续可扩展：

- 达到特定错误次数后结束
- 分阶段解锁更复杂垃圾池
- 高分阈值触发难度提升

## 3. 与限时挑战的区别

限时挑战和无尽刷分看起来接近，但在设计上应明确区分：

| 维度 | 限时挑战 | 无尽刷分 |
| --- | --- | --- |
| 目标 | 时间内尽量多处理 | 时间内持续高效刷分 |
| 节奏 | 稳定刷题 | 难度逐步增加 |
| 核心成绩 | 总处理数、正确率 | 最终高分、稳定度 |
| 扩展重点 | 补货节奏 | 难度递增 |

无尽刷分的关键是“难度曲线”，否则它会和限时挑战过于相似。

## 4. 技术实现方案

### 4.1 模式控制器

建议新增：

```text
Assets/Scripts/Modes/EndlessScore/EndlessScoreModeController.cs
Assets/Scripts/Modes/EndlessScore/EndlessScoreConfig.cs
```

职责：

- 管理时间、分数和难度阶段
- 根据阶段切换可出现垃圾池
- 根据阶段调节刷新策略
- 生成模式摘要和扩展统计

### 4.2 难度阶段设计

首版建议采用“离散阶段”而不是连续复杂公式。

例如：

- 第 1 阶段：只出基础垃圾
- 第 2 阶段：加入补充垃圾
- 第 3 阶段：提高其他垃圾和有害垃圾占比

配置上可定义：

- 每个阶段持续多久
- 每个阶段的垃圾池
- 每个阶段的活跃垃圾数量

这样比运行时复杂权重系统更容易调试。

### 4.3 生成逻辑

可复用与限时挑战相似的生成器，但建议在无尽刷分中单独封装一层“难度驱动”：

- 模式控制器判断当前阶段
- 生成器根据当前阶段使用不同候选池
- 每次补货时都从当前阶段池中随机抽取

推荐新增：

```text
Assets/Scripts/Modes/EndlessScore/EndlessScoreSpawner.cs
Assets/Scripts/Modes/EndlessScore/EndlessDifficultyStage.cs
```

## 5. 配置方案

建议配置字段：

- `timeLimitSeconds`
- `baseActiveGarbageCount`
- `difficultyStages`
- `scorePerCorrect`
- `penaltyPerWrong`
- `stageProgressionMode`

其中 `difficultyStages` 每项可包含：

- `stageName`
- `startSecond`
- `availableGarbageItemIds`
- `activeGarbageCount`

首版不要做太复杂的 editor 工具，先让配置结构能在 Inspector 里直接编辑即可。

## 6. UI 与结算

### 6.1 HUD

HUD 至少显示：

- 剩余时间
- 当前得分
- 当前阶段或难度提示
- 当前反馈

可选：

- 当前连续正确数
- 当前处理总数

如果加入阶段提示，文案要短，例如：

- `阶段 1：基础垃圾`
- `阶段 2：混合垃圾`
- `阶段 3：易混淆挑战`

### 6.2 结算

结果页重点展示：

- 模式名：无尽刷分
- 最终得分
- 正确率
- 总处理数
- 达到的最高阶段
- 错误最多垃圾
- 最高连续正确次数

如果首版来不及做连击，也至少要在文档中预留该统计字段。

## 7. 数据与展示扩展

建议新增统计：

- `HighestDifficultyStage`
- `HighestCombo`
- `MostMistakenItemName`
- `ProcessedPerMinute`

这些数据非常适合答辩展示，因为它们可以说明：

- 玩家不仅能玩
- 模式本身具有渐进难度
- 用户在高压场景下的稳定性可以被量化

## 9. 分支与改动边界

推荐分支：

```text
codex/feature-v1-endless-score
```

主要改动区域：

- `Assets/Scripts/Modes/EndlessScore/`
- 少量 `Assets/Scripts/UI/`
- 少量 `Assets/Scripts/Analytics/`
- 少量 `Assets/Scripts/Core/`

注意：

- 不要直接在一个控制器里兼容限时挑战全部逻辑
- 不要引入和标准闯关耦合的“关卡切换”概念

## 10. 联调注意事项

- 阶段切换不能导致当前场景已存在垃圾丢失引用
- 新旧阶段之间只影响后续补货，不强制替换当前所有活跃垃圾
- 如果扩展结果页字段，要确保其他模式没有这些字段时也能正常显示

## 11. 验收标准

1. 可以正常进入无尽刷分模式。
2. 开局能生成垃圾并持续补充。
3. 难度阶段会随时间或规则提升。
4. 结算页能显示高分、最高阶段和错误热点。
5. 模式运行过程中不会破坏现有分类事件链路。
